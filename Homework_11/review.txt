1. Ссылки на DOM привязаны к тегам разметки. Если верстальщик
	поменяет какой-то тег, все сломается. Потому нужно использовать id.

2. Все глобальные переменные со ссылками на dom дерево
	(getElementById) объявляются сначала скрипта.

3. Сделать массив объектов settings, элементами которого являются
	объекта с тремя свойствами: element, message и action. element -
	ссылка на элемент DOM дерева, message - сообщение, которое выводится
	в prompt, action - стрелочная функция, которая принимает element, текст
	сообщения и выполняет целовое действие по стилизации страницы. 
	Для action ссылок можно сделать выше фабрику (функцию, которая собирает функции),
	чтобы не писать один и тот же код три раза.

4. Перебирать settings циклом и последовательно получать сообщение и
	выполнять action-ы.
